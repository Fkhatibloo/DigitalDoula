# Contributing to Digital Doula

Thank you for wanting to make Digital Doula better. 💛

## Getting Started

1. Fork the repo
2. `git checkout -b feature/your-feature-name`
3. Make your changes with care
4. Submit a PR with a clear description

## Design Principles

- **Empathy first.** Every feature should serve someone in a vulnerable, meaningful moment.
- **Simplicity.** This is not a power tool. It's a gentle, beautiful space.
- **Privacy.** User stories are personal and sacred. Handle data accordingly.

## Code Style

- TypeScript everywhere
- Descriptive variable names
- Comments for anything non-obvious
